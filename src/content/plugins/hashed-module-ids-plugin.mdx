이 플러그인은 해시를 모듈의 상대경로를 기반으로 해 모듈 ID로 4개의 문자열이 생성됩니다. 프로덕션에 사용할 것을 제안합니다.

```js
new webpack.ids.HashedModuleIdsPlugin({
  // 옵션...
});
```

## Options

이 플러그인은 다음 옵션을 지원합니다:

- `context`: 이름을 만들기 위한 컨텍스트 디렉터리 (**절대 경로**).
- `hashFunction`: 사용할 해싱 알고리즘, 기본 값은 `'md4'`입니다. Node.JS의 [`crypto.createHash`](https://nodejs.org/api/crypto.html#crypto_crypto_createhash_algorithm_options)의 모든 기능이 지원됩니다.
- `hashDigest`: 해시를 생성할 때 사용할 인코딩, 기본 값은 `'base64'`입니다. Node.JS의 [`hash.digest`](https://nodejs.org/api/crypto.html#crypto_hash_digest_encoding)의 모든 인코딩이 지원됩니다.
- `hashDigestLength`: 사용할 해시 다이제스트의 접두사 길이, 기본 값은 `4`입니다. 생성된 일부 ID는 모듈 ID 충돌을 피하기 위해 여기에서 지정된 것보다 길 수 있습니다.

## Usage

다음은 이 플러그인을 사용하는 방법의 예시입니다:

```js
new webpack.ids.HashedModuleIdsPlugin({
  context: __dirname,
  hashFunction: 'sha256',
  hashDigest: 'hex',
  hashDigestLength: 20,
});
```
